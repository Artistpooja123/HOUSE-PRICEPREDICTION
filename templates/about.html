About the project

The machine learning behind the estimate

A transparent look at the dataset, the algorithms, and how the final model was chosen.

🧠

What is Machine Learning?

Machine learning lets a program learn patterns from historical examples instead of following hand-written rules. Here, the model studies past house sales to learn how features like area and location relate to price, then applies that pattern to new houses.

📐

Linear Regression, in plain terms

Linear Regression fits a straight-line relationship between the inputs and the price — useful as a baseline, but it struggles when the real relationship has curves or interactions between features.

🌳

Decision Trees & Random Forests

A Decision Tree splits the data into branches based on feature thresholds (like "Area > 2000 sq ft?"). A Random Forest trains many such trees on random subsets of data and averages their predictions, which usually improves accuracy and reduces overfitting.

🗂️

The dataset

House_Price_Dataset.csv contains {{ '{:,}'.format(1200) }} synthetic-but-realistic property records with 12 features: area, bedrooms, bathrooms, floors, parking, year built, location, condition, and nearby schools, hospitals, and metro access.

Workflow

Project workflow, end to end

Data collection

A structured dataset of property features and sale prices is assembled into a CSV file.

Preprocessing

Missing values are filled, categorical fields are label-encoded, and numeric fields are standardized.

Model training

Three regressors are trained on the processed training split.

Evaluation

Each model is scored with MAE, MSE, RMSE, and R² on unseen test data.

Serialization

The winning model and its scaler/encoders are saved to disk with pickle.

Deployment

Flask loads the saved files and exposes a form-driven prediction endpoint.
